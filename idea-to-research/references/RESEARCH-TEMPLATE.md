# Research Document Template

> Use this template as the structure for all research documents produced by the `idea-to-research` skill.

---

## Output Filename Pattern

```
{project_root}/docs/research/{YYYY-MM-DD}-{feature-slug}-research.md
```

Example: `docs/research/2025-01-15-real-time-collaboration-research.md`

## Section Guidelines

### Executive Summary
- Keep it scannable (bullet points)
- Lead with key recommendations
- Highlight decisions needing input
- Include effort estimation

### Intent & Requirements
- Be specific about scope
- User stories should be testable
- Clearly separate MVP from future

### Codebase Analysis
- Include actual file paths
- Show code snippets for patterns
- Map all integration points

### Technology Research
- Compare at least 2-3 options
- Use consistent scoring
- Include code examples

### Architecture Blueprint
- Visual diagrams are valuable
- Define all data models
- Specify API contracts

### Implementation Considerations
- Be honest about complexity
- Identify all risks
- Include test strategy

### Decisions
- Every significant decision documented
- Options with trade-offs
- Clear recommendations

### File Reference Map
- All paths must exist
- Include line numbers where possible
- Group by action type

### Open Questions
- Don't hide unknowns
- Be clear about impact
- Suggest resolution path
